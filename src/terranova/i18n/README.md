# Translations

Native Qt strings are translated via Qt's `.ts` files (one per locale).  The
web tier uses `react-i18next` against the same string catalogue, generated by
the release workflow.

## Adding a locale

```bash
pylupdate6 src/terranova/**/*.py -ts src/terranova/i18n/fr.ts
# edit fr.ts in Qt Linguist
lrelease src/terranova/i18n/fr.ts
```

Supported locales for v1.0:

- `en_GB` (default — no .ts needed)
- `fr`
- `es`
- `pt`
- `de`
- `it`

Crowdin integration ships in Phase 4.
